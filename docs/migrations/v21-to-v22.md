# NGX-MORYX-WEB 21.x to 22.x

This migration guide covers the migration from version 21.x to 22.x of the ngx-moryx-web framework.

## Prerequisites

Before upgrading, ensure your application is running the latest 21.x version without any deprecation warnings.

## Dependency Updates

### Angular 21 to 22

Update all Angular packages to version 22. For detailed information about Angular-specific changes, breaking changes, and upgrade commands, refer to the [Angular 22 Update Guide](https://angular.dev/update-guide).

### ngx-translate 17 to 18

Update ngx-translate packages to version 18. For detailed information about ngx-translate-specific changes and upgrade instructions, refer to the [ngx-translate Release Notes](https://github.com/ngx-translate/core/releases).

## Visual Changes

### Angular Material Density Reset

The Angular Material density setting has been reset from `-4` to `0` (default density).

Components will appear larger with more padding and spacing. This includes form controls, buttons, and input fields. The UI will feel less compact, which improves touch target sizes and accessibility.

If you need a more compact layout, override this in your custom theme:

```scss
@include mat.theme((
  color: ( /* your color config */ ),
  typography: Roboto,
  density: -4, // or any value between -5 and 0
));
```

## Breaking Changes: SCSS Variables Removed

The following SCSS variables have been removed from the framework and must be replaced with Angular Material 3 global CSS variables:

### Variable Replacement Table

| Old Variable (v21) | New CSS Variable (v22) | Usage Context | Description |
|-------------------|------------------------|---------------|-------------|
| `$mat-background` | `var(--mat-sys-surface)` | Background colors | Main application background surface |
| `$mat-background-light` | `var(--mat-sys-surface-container-lowest)` | Component backgrounds | Light surface for cards, panels, containers |
| `$mat-color` | `var(--mat-sys-on-surface)` | Text/foreground | Primary text color on surfaces |
| `$mat-color-light` | `var(--mat-sys-surface-container-low)` | Light text colors | Secondary/light text colors |
| `$mat-color-caption-light` | `var(--mat-sys-outline)` | Caption/subtitle text | Muted text for captions and subtitles |
| `$lighter-background-hover` | `var(--mat-sys-surface-container-low)` | Hover states (subtle) | Subtle hover background for light interactions |
| `$light-background-hover` | `var(--mat-sys-surface-container-highest)` | Hover states (prominent) | Prominent hover background for lists/items |

### Migration Steps for SCSS Variables

1. Remove the variables import from your SCSS files:

   ```scss
   // Remove this line:
   @use "@moryx/ngx-web-framework/styles/variables" as *;
   ```

2. Replace all variable usages according to the table above. Examples:

   ```scss
   // Before (v21):
   .my-component {
     background-color: $mat-background-light;
     color: $mat-color;
   }

   .my-list-item:hover {
     background-color: $light-background-hover;
   }

   // After (v22):
   .my-component {
     background-color: var(--mat-sys-surface-container-lowest);
     color: var(--mat-sys-on-surface);
   }

   .my-list-item:hover {
     background-color: var(--mat-sys-surface-container-highest);
   }
   ```

The new CSS variables follow Material Design 3's semantic elevation system:

- `--mat-sys-surface` - Base surface (main background)
- `--mat-sys-surface-container-lowest` - Lowest elevation container
- `--mat-sys-surface-container-low` - Low elevation container
- `--mat-sys-surface-container` - Medium elevation container
- `--mat-sys-surface-container-high` - High elevation container
- `--mat-sys-surface-container-highest` - Highest elevation container (hover states, selected items)
- `--mat-sys-on-surface` - Text/content on surfaces
- `--mat-sys-outline` - Borders, dividers, caption text

## UI Improvements

EntryEditor improvements:

- Fixed layout inconsistencies in entry editor components
- Added padding around controls for better visual hierarchy
- Description hints are now only shown when actually present
- Dynamic subscript sizing: Form fields now use `subscriptSizing="dynamic"` to prevent layout shifts
- Entry object buttons are now properly aligned
- Replaced custom icon spans with Material icons

These changes don't require any code changes in consuming applications.

## Migration Steps

1. Update dependencies (Angular, ngx-translate - see sections above)
2. Migrate SCSS variables in your stylesheets (see Breaking Changes section)
3. Test your application, especially forms, entry editors, translations, and custom styled components
4. Review and adjust layouts if needed (density changes may affect spacing)
